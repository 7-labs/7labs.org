# Growth Plan for English Users

## Core principle

7labs.org should not become a generic AI blog. Every entry page must include a working tool, sample input, sample output, and a next action.

## 30-day goals

- Launch and maintain 32 working tools.
- Get Google Search Console and Bing indexing started.
- Add structured example runs, sample output, limitations, and upgrade path to every tool page.
- Maintain 5 Best AI for pages and 4 comparison pages with last-reviewed dates and selection criteria.
- Add no-op analytics events for tool runs, copy actions, feedback, category filters, and best/compare clicks.
- Keep newsletter capture behind `NEWSLETTER_PROVIDER`, with `none` as the launch default.

## 90-day goals

- 50+ tools only after the first 32 have usage data and clear conversion paths.
- 50+ reviewed tool, prompt, comparison, and model pages before attempting larger programmatic expansion.
- 500+ newsletter subscribers.
- Identify 1-3 repeat-use tools.
- Start paid tests for document workflows.

## English-market channels

- Product Hunt: launch AI Tool Finder.
- Hacker News: share the developer tools and technical build notes.
- Reddit: post use-case-specific tools where allowed.
- Indie Hackers: share progress and metrics.
- Dev.to: publish developer utility tutorials.
- GitHub: open-source selected utility tools to earn developer trust.
- YouTube/TikTok: publish short workflows using Prompt Studio and Creator Lab.
- LinkedIn: publish B2B workflows and tool comparisons.

## SEO page types

1. Tool pages: `/tools/regex-generator`
2. Best-of pages: `/best/best-ai-for-coding`
3. Comparison pages: `/compare/chatgpt-vs-gemini`
4. Prompt pages: `/prompts/product-photo`
5. Model pages: `/models/gemini-flash-lite`
6. Tutorials: `/blog/how-to-use-ai-for-x`

## Content requirements

- Put the usable tool at the top.
- Do not publish empty AI-generated directory pages.
- Include sample input and output.
- Mention limitations and caveats.
- Update pages that mention third-party prices, features, or quotas.
- Add first-hand testing notes when recommending a paid tool.
- Keep FAQ schema aligned with visible FAQ content only.
- Add BreadcrumbList schema to tool, best-of, and comparison pages.
- Keep sitemap dates stable unless a page is materially reviewed.
- Keep `/best` and `/compare` hub pages current so crawlers and users can discover every guide without relying on query filters.
